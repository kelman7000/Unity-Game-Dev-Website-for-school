Comments

There are two types of comments in the C# language.

  1. Single line comment - //
  2. Multi-line comment - /**/

Comments are not executed at run time, they are simply there to make the code easier to understand, and are most helpful to those that did not write the program. A good comment will tells us what a class or method does, but not how this task is accomplished.

Single Line Comment

The single line comment is used when we have somthing short to say and is best left under 80 characters

Multi-line Comment

The multi-line comment is used when we have somthing long to say that will take up more than 80 characters. Each line should be 80 characters at most. Why the 80 characters? Becuase lots of time a developer will have to file tabs open in their IDE or text editor and this will prevent them form having to side scroll, trust me it's annoying.

Variables

Variables are somthing that store information that is used during to execution of the program. As the name suggests the information stored can change thought execution hence the vari, but there are others that work the same way but remain constant throught the entire execution these are called constants.

All variables & constants have a type, this sepcefies what kind of data can be stored. Here we will go over the ones you'll be using.

int:

This type is for storing while numbers i.e. [1 ,2 ,3 ,4 ,5 ,6 ,7 ,8 ,9 0]

double:

This type is able to store fractional number up to 15 digits.

float:

This also stores fractional numbers but only up to 6-7 digits.

char:

This type is used for storing a single character (NOTE: CAN BE ASSIGNED ASCII CHARACTER CODES THROUGHT CASTING).

string:

Used to store a series of characters, this could be a single word or a sentance (NOTE: IF THE VALUE IS GOING TO BE CHANGING DURING EXECUTION USE STRING BUILDER).

boolean:

This type can be one of two values, either true or false.

Casting:

Casting is when we explixetly convert one type to another.Though you note that this does not work for all type conversions.

Loops

loops are used when we want to execute some block of code multiple times before moving on.

for loop

for loops are usful when dealing with arrays and you need the index of each element.

foreach loop

foreach loops are also usful when working wiht arrays but the index of each element is not needed i.e. you want to go over every element.

Basically just a different way of writting for loops, but they are different and one is better than the other depenting on the situation.

These function as while loops but they are garented to execute at least one time. Everything in the do block is executed and then the continuation conditional is checked.

Objects

An object is an instance of a class, they use the class as a blueprint to create the object. Objects can be create in any number of ways using different types of constructors. A class can have more than one constructor but each constructor must be different from the other(s). As C# is an OOP language this is were all its strength comes from. OOP is a programming model that organizes everything into objects, like in the real world we have cars, computers and much more. Different objects can work together to accomplish some task i.e. the wheels of a car work with the gas pedal and other stuff to make the car move.

Coroutines

Coroutines are a special type of method, there are a few major differences though. One they are defined with IEnumberator were there would otherwise be a return type, two, when a method is executed in Unity it does all its operations in the frame it was called. For example if you wanted to change the color of an object over some time interval you would not be able to use a function becase everything happends in one frame. With a coroutine however you can pause the execution return and then continue where you last left off in the new frame.

Inheritance

You are all propably awake of how people take on traits from their parents i.e. their hair color, eye color, height, faceial features...Well that is called inheritance and it can be achived in many programming languages although it is not exactly the same as how it with people. Say for example we want to make a game about pets on a farm, we could have a dog, cat, cow, chicken and so on. All these animals need to eat, sleep, walk, make a sound, have some number of legs, either have a tail or don't. They will also all share some attributes, now one thing we can do is write all this out for all the different animals or we could create on class called Animal and write methods for walking....and define attributs like number of legs. Then create classes for different animals like Dog, Cat and so on, these classes can then inherite from the Animal class and have all the method and attribues there already.

Polymorphism

This goes hand in hand with Inheritance, as we created the Animal class that has a method makeSound() then made all the animals (Dog, Cat,...) inherite this method makeSound(). But as we all know a dog and a cat do not make the same sound, so in the Dog class we can override the makeSounds() method to be a dog sound and do the same with the Cat class to make it say a cat sound.

Best Practice

Here I will go over some of the best practices for C# programming.

  • Constant variables should be all caps
  • Use meaningful names (classes, methods, variables)
  • use Camel case for variables i.e. spawnInterval
  • use Pascal for methods and classes i.e. ClassName
